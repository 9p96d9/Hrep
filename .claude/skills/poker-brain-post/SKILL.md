# /poker-brain-post — AI解析出力レビュー

## このSKILLの仕事

AI解析出力を `specs/ai_analysis.md` の品質ルールに照らし合わせてチェックする。
問題があれば具体的な修正箇所をレポートする。対象は `detail_street` 形式を主とする。

---

## 品質チェックリスト（実行順）

### チェック1: 主語ルール
- [ ] 全フィールドで主語がポジション名（CO/BB/BTN等）になっているか
- ❌ 「ヒーローはコールした」 → ✅ 「BBはBTNのCBetに対してコールした」

### チェック2: MDFの使用場所（detail_streetは構造で検証）
- [ ] "MDF" が `river_analysis` **のみ**に含まれるか
- [ ] `preflop_context` / `flop_read` / `turn_read` / `opp_exploit` / `kaizen` に "MDF" がないか
- **修正**: 該当フィールドのMDF言及を削除し、エクイティ・ドロー・レンジで書き直す

### チェック3: ストリート到達の整合
- [ ] ハンドが到達していないストリートのフィールドが空文字 `""` になっているか
- [ ] river到達ハンドの `river_analysis` が非空で "MDF" or "必要エクイティ" を含むか

### チェック4: 数値の出典
- [ ] [GTO数学]ブロックの値以外の数値（bb・%）が含まれていないか
- [ ] AIが自前で計算した数値（根拠のないパーセンテージ等）がないか
- **修正**: 数値を削除するか、[GTO数学]ブロックの値に置き換える

### チェック5: レンジ表記
- [ ] `preflop_context` に具体的なハンド表記（例: ATo+, KQs, 22-44）が含まれるか
- [ ] 「強いハンド」「弱いドロー」等の曖昧表現が避けられているか

### チェック6: 評価軸
- [ ] 「良い/悪い/ミス」という結果論的評価が含まれていないか
- [ ] 「このスポットでHeroのレンジは均衡しているか」を軸にしているか

### チェック7: opp_exploit の具体性
- [ ] アクション名（"bet", "raise", "fold" 等）を含む具体的な搾取戦略になっているか

---

## レポート形式

```
✅ チェック1 主語ルール: OK
❌ チェック2 MDF使用場所: flop_read でMDF言及あり（修正箇所: 2文目）
✅ チェック3 ストリート到達: OK
⚠️ チェック5 レンジ表記: preflop_contextが曖昧（「強いハンド中心」→具体化が必要）
...
修正が必要な箇所: 2件
```

---

## $ARGUMENTS の使い方

引数なし → 最後にレビューしたAI出力を対象
`hand_id` 指定 → その特定ハンドの解析のみをチェック
`--fix` フラグ → 問題を自動修正して書き直し（実験的）
