# /poker-brain-pre — 解析前スポット分析

## このSKILLの仕事

AI解析を実行する前に「このスポットで何が起きているか」を正確に把握する。
AIへの入力品質を上げることで、出力品質を上げる。

---

## スポット分析チェックリスト

### Step 1: ポジション確認
- [ ] HeroはIP（ポストフロップで後からアクション）か OOP（先にアクション）か
- [ ] 判定方法: 各ストリートの最初にアクションするプレイヤー = OOP
- [ ] BTNは常にIP（例外なし）
- [ ] SB/BBはフロップ以降でOOP（BTN以外のプレイヤーに対して）

### Step 2: レンジ優位の確認
- [ ] このボードはHeroのレンジに有利か不利か
- [ ] フロップ: 高いカード（K/A）はプリフロップレイザーのレンジに有利
- [ ] フロップ: 低いカード（2/3/5）はBBのデフェンスレンジに有利
- [ ] ターン: 追加カードがレンジをどう変えるか

### Step 3: ポットタイプの確認
- [ ] シングルレイズポット（SRP）か3betポットか
- [ ] 3betポットではレンジが絞られ、ブラフの成功率が変わる
- [ ] ポットサイズ（bb）とスタック深度（残りスタック÷ポット = SPR）

### Step 4: このスポットのカテゴリ確認
`specs/classify.md` を参照してカテゴリを確認する（11カテゴリ・nice_call/bad_call含む）。

### Step 5: GTO数学の事前計算
`specs/gto_math.md` を参照して、このスポットのαとMDFを事前把握する。
- MDF を言及するのはリバーのみ
- フロップ/ターンはエクイティ・ドロー・レンジで論じる

## プリフロップ想定レンジ（目安）

| ポジション | RFI目安 | 代表ハンド |
|---|---|---|
| UTG | ~13% | AA-77, AKo-AJs, KQs |
| HJ | ~17% | +A9s, KJs, QJs |
| CO | ~25% | +A8s, KTs, QTs, JTs |
| BTN | ~42% | +A2s, K9s, Q9s, J9s, 98s |
| SB | ~38% vs BB | 幅広いがSBの不利な位置を考慮 |
| BB | 全ポジションのRFIに対してdefend | ポジション不利分を調整 |

> レンジ表のJSONデータ資産化はV2候補（REQUIREMENTS.md「V2以降の検討候補」参照）。
> 現時点ではこの表が唯一のレンジ参照。存在しないファイルを参照しないこと。

## $ARGUMENTS の使い方

引数なしで起動した場合 → このチェックリスト全体を実行
引数あり（例: `CO river bet-call`）→ そのスポット特有の注意点に絞って分析
