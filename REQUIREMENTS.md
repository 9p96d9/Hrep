# HrepNext — プロダクト要件定義

## プロダクトの約束

> 勝ったか？ではなく、上手かったか？を答える装置。

多くのポーカーツールは結果を説明する。HrepNextは結果から切り離された実力を証明する。

---

## ターゲットユーザー

**第一弾（橋頭堡）：**
すでにGTO Wizardやソルバーに課金していて（＝支払い意思とGTO信仰は実証済み）、しかし「ハンドは後で見る」と言って、その"後で"が永遠に来ない人。

実証済みの意図 × 実証済みの摩擦 = 最初に解くべき問題。

---

## ユーザーストーリー

### Must（必須）

- プレイ後にアプリを開いたとき、最初に見るものが「今日の最も優れた判断」である
- 難易度と正しさに基づいたナイスプレイが先頭に表示される
- **GTO的に正しいコールは、ショーダウンで負けていてもナイスプレイ対象である（`nice_call`）。結果と判断の分離はフォールド側だけでなくコール側にも適用する**
- **ナイスプレイの直後に「改善チャンス」を同じ客観トーンで表示する。対象はGTO判定で不正解が確定したスポット（bad_fold / bad_call）のみ・最大3件・0件も正直に表示**
- 難易度スコアは「カテゴリの定義上の難しさ × ストリート」で計算され、結果（損益）を含まない
- AI解析はストリートごとに展開され、主語はポジション名（「CO raises」「BB calls」）である
- AI解析は[GTO数学]ブロックの数値のみ引用し、AIが自前で計算した数値を使わない
- MDFはリバースポットのみで言及する。フロップ/ターンでは言及しない
- ナイスプレイ該当ゼロのセッションは「0件」を正直に表示する（セクションを非表示にしない）

### Should（推奨）

- AI解析がハンドレンジを具体的に表記する（例: ATo+, KQs, 22-44）
- EV推定が解析に含まれる
- プリフロップの想定レンジを内部参照できる
- プレイ後24時間以内の再オープン率を主要KPIとして追跡する
- 難易度補正後の意思決定正答率が週/月単位で可視化される

### Must Not（禁止事項）

- **活動量（セッション数・ストリーク・開いた回数）で良い気分を稼がせない**
- **正しくないスポットを「ナイスプレイ」として称えない**（誤称賛は信頼を破壊する。ショーダウンで勝っていてもGTO判定が不正解なら称えない）
- **判定が確定していないスポットを「改善チャンス」として指摘しない**（誤指摘も誤称賛と同じく信頼を破壊する。迷ったらwarnに倒す）
- **おべっかマシンになる**（毎セッション必ずナイスプレイを出し続けるな）
- AI解析で「良い/悪い/ミス」という結果論的評価を出さない
- 結果（損益）をナイスプレイの判断材料にしない
- Stravaのように「可視化するだけ」で終わる——ポーカーは環境が嘘をつく。真実を立てることが仕事

> **SPECとの整合ルール:** 上のMust/Must Notは `specs/` の受入基準より優先する。
> 受入基準がここと矛盾していたら受入基準の方がバグである（Hrepでは「0件→セクション非表示」という矛盾した受入基準が混入していた）。

## 指標設計

### 先行指標（習慣戦に勝っているか）
**プレイ後24時間以内の再オープン率**
競合は「振り返らない自分」。ここが日々の戦線。

### 遅行指標（勝つに値するか）
**難易度補正後の意思決定正答率の曲線**
数週間〜数ヶ月で上昇していく曲線。これが本物の成長の証明。

## V1 スコープ

| 機能 | 方針 |
|---|---|
| ハンドキャプチャ（Chrome拡張） | 継続 |
| GTO分類 — **11カテゴリ**（`nice_call`/`bad_call` 新設） | **拡張**（`specs/classify.md`） |
| GTO数学計算（α/MDF/難易度） | 継続 |
| 高難度正解セクション（bluff_catch/nice_call/nice_fold） | **拡張**（nice_call追加） |
| **改善チャンスセクション（bad_fold/bad_call）** | **新規**（`docs/features/improve_chances.md`） |
| AI解析 — `detail_street` モードを正式版とする | **本命**（旧 `detail` は移行期間のみ併存） |
| AIプロバイダー | Groq/Gemini BYOK（現状維持）。Claude Fable 5 はSPEC外の品質実験のみ |
| MDF = リバーのみ | 継続 |
| PDF出力 | 削除（V2以降で判断） |

## スコープ外（V1で作らない）

- PDF生成（WeasyPrint）
- classified_snapshot（gzip+base64）— Firestore移行で不要化
- PostgreSQL / AWS構成 — 2026-06にCloud Run + Firestoreへ移行済み。再作成しない
- 対戦相手統計DB・SNS共有（旧Phase 11）

## V2以降の検討候補（今回見送り・アイデアの記録）

- **KPIの担当機能**: 先行KPI（24h再オープン率）を動かすコンテンツ駆動通知
  （「今日の最高難度スポットが1件ある」——ストリークでなく中身で呼び戻す）と、
  遅行KPI（難易度補正後の正答率曲線）を計算・保存・表示する週次集計。
  現状、両KPIとも計測するだけで動かす機能が存在しない
- **3betポット難易度補正**: `is_3bet_pot` はデータに存在するが難易度スコアが未使用
- **誤称賛報告ボタン**: ナイスプレイカードに「この判定は間違い」→ 誤称賛率を品質KPI化
- **preflop_ranges のデータ資産化**: レンジ表をJSONで持ち、AIプロンプトとUIの両方から参照
  （`/poker-brain-pre` が参照を約束している `docs/preflop_ranges.md` は未作成のまま）
