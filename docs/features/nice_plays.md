# docs/features/nice_plays.md — 高難度正解セクション

**種別: UI仕様（テストなし）**
**関連SPEC:** `specs/gto_math.md`（スコア計算の定義）

---

## 設計哲学

> ポジティブ先頭は、硬い真実を人間に届ける唯一の配送手段である。

最初に「今日の最も優れた判断」を厳密に（おべっかでなく）提示する。
それが感情的なクレジットを生み、続く「改善チャンス」も厳密に読ませる。

---

## 表示条件

- セクションは**常に表示**する
- `nice_play_score ≥ 0.5` かつカテゴリが `{bluff_catch, nice_call, nice_fold}` のハンドを最大3件（スコア降順）
  - `bluff_catch` はGTO判定=不正解（幸運なコール）なら除外（`specs/classify.md` §4）
- 条件を満たすハンドが0件 → **「今日のナイスプレイ: 0件」を正直に表示**（セクションを非表示にしない。REQUIREMENTS.md Must）

### nice_call カードの特記

`nice_call`（負けたが正しかったコール）は損益がマイナスのまま先頭セクションに載る。
これがこのプロダクトの約束の最も強い実演なので、カードに客観表記の一行を添える:
「必要エクイティを満たすコール。損失は分散」——称賛語は使わず、事実だけを書く。

## 配置

サマリーバーの直下・タブより上。ページを開いた瞬間に視野に入る位置。

```
[サマリーバー]
[EV差分表示] ← ある場合のみ
[高難度正解セクション] ← ここ（常に表示・0件でも表示）
[改善チャンスセクション] ← 直後に配置（docs/features/improve_chances.md）
[タブバー]
```

---

## 各カードに表示する情報

| 要素 | 内容 |
|---|---|
| ランク | #1 / #2 / #3 |
| ハンド番号 | H23 |
| カテゴリバッジ | 「ブラフキャッチ」「ナイスフォールド」 |
| ストリート | リバー / ターン / フロップ |
| ポジション | Hero vs 相手（例: BB vs BTN） |
| 手札 | スートカラーリング付き |
| 難易度スコア | ●●●●○（5段階の●/○表記） |
| 損益 | +X.Xbb / -X.Xbb（参考値・色分け。**ランク順位には使わない**） |

---

## インタラクション

- カードをクリック → 対象ハンドカードにスクロールジャンプ（`id="hcard-{hnum}"`）
- セクションタイトル横のサブテキスト: 「分散と切り離した判断品質 — bluff catch / nice call / nice fold スポットを難易度順に表示」

---

## トーン設計

- タイトル: 「高難度正解」（おべっかではなく、事実の記述）
- 称賛文は使わない（「よくできました」「すばらしい」等は禁止）
- スコア・カテゴリ・ストリートという客観データで自己評価させる
