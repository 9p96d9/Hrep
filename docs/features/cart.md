# docs/features/cart.md — AI解析カート

**種別: UI仕様（テストなし）**
**関連SPEC:** `specs/ai_analysis.md`（解析モード・品質ルール）

---

## 概要

解析結果画面でハンドを選択してAI解析を実行するフロー。
BYOK（Bring Your Own Key）方式でユーザーのAPIキーを使用。

---

## カートの動作フロー

```
1. ハンドカードの「🛒」ボタンをクリック → カートに追加（青枠・バッジ更新）
2. FABまたはフッターの「解析カート」をクリック → カートパネルを開く
3. APIキーを入力（gsk_... → Groq / その他 → Gemini）
4. 「⚡ 解析を実行」ボタン → SSEストリームで進捗表示（detail_streetモード）
5. 完了 → 各ハンドカードにAI解析結果がインライン表示
```

- `preflop_only` ハンドはカート追加不可（分析素材不足）
- 再解析: 同じハンドを再送した場合は上書き

---

## APIキー管理

- 入力されたキーはセッションストレージに保存（ブラウザリロードで消える）
- `gsk_` で始まる → Groq (llama-3.3-70b-versatile)
- それ以外 → Gemini (gemini-2.5-flash)
- キーはサーバーに保存しない（BYOKフロー）

---

## トークン見積もり

解析実行前に概算トークン数を表示する。
- バッチ解析（detail_street mode）: カート件数 × 約400 tok
- 詳細解説（explain mode）: 1ハンドあたり約1,700 tok

---

## カートパネルUI

| 要素 | 説明 |
|---|---|
| カートアイテム | ハンド番号・ポジション・損益 |
| 削除ボタン | 個別ハンドをカートから除去 |
| トークン見積もり | 解析前のコスト概算 |
| APIキー入力 | password型（非表示） |
| 解析実行ボタン | カートが空または未認証時は disabled |
