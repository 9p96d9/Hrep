# docs/features/improve_chances.md — 改善チャンスセクション

**種別: UI仕様（テストなし）**
**関連SPEC:** `specs/classify.md`（bad_fold / bad_call の判定）, `specs/gto_math.md`（スコア計算）

---

## 設計哲学

> ポジティブ先頭は配送手段。このセクションが**荷物**である。

高難度正解セクションが生んだ感情的クレジットで、硬い真実——「このスポットは判断を間違えた」——を
読める状態のユーザーに届ける。だからこのセクションは必ずナイスプレイの**直後**に置く。

信頼の対称性: ナイスプレイで誤称賛しないのと同様に、ここでも**誤指摘しない**。
GTO判定で不正解が**確定**したスポットだけを載せる（warn系・bluff_failedは載せない）。

---

## 表示条件

- セクションは**常に表示**する（高難度正解セクションと同じ扱い）
- 対象カテゴリ: `{bad_fold, bad_call}` のみ
  - `bluff_failed` は対象外——ブラフは混合戦略上、一定頻度で失敗するのが均衡であり「間違い」ではない
  - `call_lost` / `fold_unknown`（warn）は対象外——判定が確定していないものを指摘しない（Must Not）
- 最大表示件数: 3件
- 0件 → **「今日の改善チャンス: 0件」を正直に表示**（セクションを非表示にしない）

## 表示順序

```
優先度 = ストリートウェイト降順（river → turn → flop）、同ストリート内はポット額(bb)降順
```
理由: リバーの間違いはエクイティ確定後の純粋な判断ミスで、最も学習価値が高い。
難易度スコア順にしないのは、bad系の基礎スコアが一律0.30で順位がつかないため。

## 配置

```
[サマリーバー]
[EV差分表示] ← ある場合のみ
[高難度正解セクション] ← 常に表示
[改善チャンスセクション] ← ここ（常に表示・0件でも表示）
[タブバー]
```

## 各カードに表示する情報

| 要素 | 内容 |
|---|---|
| ハンド番号 | H23 |
| カテゴリバッジ | 「バッドフォールド」「バッドコール」 |
| ストリート | リバー / ターン / フロップ |
| ポジション | Hero vs 相手（例: BB vs BTN） |
| 手札 | スートカラーリング付き |
| GTO根拠 | [GTO数学]の該当値をそのまま表示（例: 「MDF基準67%に対しフォールド」） |
| 損益 | +X.Xbb / -X.Xbb（参考値） |

> ランク表示（#1/#2/#3）は付けない。間違いに順位をつけて競わせない。

---

## インタラクション

- カードをクリック → 対象ハンドカードにスクロールジャンプ（`id="hcard-{hnum}"`）
- カード内に「AI解析カートへ追加」ショートカット（改善チャンス → 深掘りの導線）
- セクションタイトル横のサブテキスト: 「GTO判定で確定した改善余地 — 数値根拠付き」

---

## トーン設計

- タイトル: 「改善チャンス」（「ミス」「悪いプレイ」という結果論・裁きの語は使わない）
- 説教文は使わない。GTO根拠の数値（MDF基準・必要エクイティ）だけを提示し、自己評価させる
- 「惜しい」「もったいない」等の感情語も禁止——高難度正解セクションと同じ客観トーンで対称にする
